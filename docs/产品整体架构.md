## 1. 项目愿景与目标

### 1.1 定位

Sky Alpha Pro 是一个专注于 **Polymarket 天气预测市场** 的 AI 驱动量化交易系统。通过整合多源气象数据与 AI 分析能力，发现市场定价偏差并自动执行交易，在天气预测市场中获取超额收益。

### 1.2 核心竞争力

| 维度 | 竞争优势 |
|------|---------|
| **数据优势** | 整合 NOAA/NWS (解析源)、Open-Meteo (多模型预报)、Visual Crossing 三级气象数据体系，构建独立概率估计 |
| **AI 决策** | 基于 Google Gemini 的 Function Calling Agent，实现 "数据获取 → 分析 → 决策 → 执行" 全闭环 |
| **链上洞察** | 监控 Polygon 链上交易，识别竞争对手套利机器人行为模式 |
| **策略进化** | 追踪顶尖玩家策略并复盘对比，持续迭代自身模型 |

### 1.3 MVP 范围

**纳入 MVP:**
- 天气市场数据采集与监控（温度类市场优先）
- AI Agent 概率评估与交易信号生成
- 基础交易执行（限价单）
- 链上竞争对手监控
- CLI 工具 + REST API
- PostgreSQL 数据持久化

**不纳入 MVP:**
- Web 前端 / Dashboard
- 跨平台套利 (Kalshi/ForecastEx)
- 降水 / 飓风等非温度市场
- 全自动无人值守交易
- 历史预报偏差 ML 训练

### 4.1 AI Agent 引擎

#### 架构

采用 Google ADK 的 `llmagent` 构建核心 Agent，通过 Function Calling 实现 "感知 → 思考 → 行动" 闭环。

```

│  ┌─────────────────┐  ┌──────────────────────┐  │
│  │ get_weather_     │  │ get_market_prices    │  │
│  │ forecast         │  │                      │  │
│  └─────────────────┘  └──────────────────────┘  │
│  ┌─────────────────┐  ┌──────────────────────┐  │
│  │ get_historical_  │  │ calculate_edge       │  │
│  │ weather          │  │                      │  │
│  └─────────────────┘  └──────────────────────┘  │
│  ┌─────────────────┐  ┌──────────────────────┐  │
│  │ place_order      │  │ get_competitor_      │  │
```

#### Function Calling Tool 定义

**Tool 1: `get_weather_forecast`**
```
描述: 获取指定城市的天气预报数据
参数:
  - location (string, required): 城市名或坐标, 如 "New York, NY"
  - source (string, optional): 数据源 "nws" | "openmeteo" | "visualcrossing" | "all"
  - days (int, optional): 预报天数 1-16，默认 7
返回: 多源预报数据，包含温度、降水、风力等
```

**Tool 2: `get_market_prices`**
```
描述: 获取 Polymarket 天气市场的当前价格
参数:
  - market_slug (string, optional): 特定市场 slug
  - city (string, optional): 按城市筛选
  - active_only (bool, optional): 仅活跃市场，默认 true
返回: 市场列表，包含 YES/NO 价格、成交量、到期时间
```

**Tool 3: `calculate_edge`**
```
描述: 计算气象预测概率与市场价格之间的边际优势
参数:
  - market_id (string, required): 市场 ID
  - forecast_data (object, required): 预报数据
  - method (string, optional): 计算方法 "basic" | "ensemble" | "bayesian"
返回: 边际值 (edge)、建议方向 (YES/NO)、置信度
```

**Tool 4: `place_order`**
```
描述: 在 Polymarket 下单（需要 confirmation）
参数:
  - market_id (string, required): 市场 ID
  - side (string, required): "BUY" | "SELL"
  - outcome (string, required): "YES" | "NO"
  - price (float, required): 限价价格 0.01-0.99
  - size (float, required): 下单数量 (USDC)
返回: 订单 ID、状态
```

**Tool 5: `get_nws_observation`**
```
描述: 获取 NWS 官方气象站最新观测数据（市场解析数据源）
参数:
  - station_id (string, required): 气象站 ID, 如 "KNYC"
返回: 实时温度、湿度、风速等观测值
```

**Tool 6: `get_competitor_activity`**
```
描述: 获取链上竞争对手（套利机器人）近期活动
参数:
  - market_id (string, optional): 特定市场
  - hours (int, optional): 回看小时数，默认 24
返回: 可疑机器人地址列表、交易模式分析
```

**Tool 7: `get_player_positions`**
```
描述: 获取顶尖玩家在天气市场的持仓情况
参数:
  - player_address (string, optional): 特定玩家地址
  - top_n (int, optional): 前 N 名玩家，默认 10
返回: 玩家持仓列表、PnL、历史胜率
```

**Tool 8: `get_historical_weather`**
```
描述: 获取历史天气数据用于偏差分析
参数:
  - location (string, required): 城市名或坐标
  - start_date (string, required): 起始日期 YYYY-MM-DD
  - end_date (string, required): 结束日期 YYYY-MM-DD
  - metrics ([]string, optional): 指标列表 ["TMAX","TMIN","PRCP"]
返回: 历史气象数据时间序列
```

#### Agent 决策流程

```
1. 定时触发 / 手动触发
      │
2. Agent 调用 get_market_prices → 获取活跃天气市场
      │
3. 对每个有价值的市场:
      │
   3a. 调用 get_weather_forecast (source="all") → 获取多源预报
      │
   3b. 调用 get_nws_observation → 获取实时观测 (到期日当天)
      │
   3c. 调用 get_historical_weather → 获取同期历史数据
      │
   3d. 调用 calculate_edge → 计算市场定价偏差
      │
   3e. 调用 get_competitor_activity → 评估竞争环境
      │
4. Agent 综合分析，决定是否交易
      │
5. 如有信号 → 调用 place_order (需 confirmation)
      │
6. 记录决策日志到数据库
```

### 4.2 气象数据服务 (WeatherService)

#### 数据源层级

#### 认证机制

Polymarket CLOB 交易使用 EIP-712 签名方案:

1. 使用以太坊私钥签名消息，派生 API Key / Secret / Passphrase
2. 订单使用 EIP-712 类型化数据签名
3. 请求头携带: `POLY-ADDRESS`, `POLY-SIGNATURE`, `POLY-TIMESTAMP`, `POLY-NONCE`

#### 天气市场数据筛选

通过 Gamma API 筛选天气相关市场:
```
GET /events?tag_id=weather&active=true&closed=false
GET /markets?tag_id=weather&active=true&order=volume_24hr&ascending=false
```

### 4.4 链上分析引擎 (ChainAnalyzer)

#### 目标

在 Polygon 链上监控 Polymarket 的 CTF (Conditional Token Framework) 交易，识别套利机器人行为。

#### 核心合约地址

| 合约 | 地址 | 用途 |
|------|------|------|
| Conditional Tokens (CTF) | `0x4D97DCd97eC945f40cF65F87097ACe5EA0476045` | ERC1155 条件代币 |
| CTF Exchange | `0x4bFb41d5B3570DeFd03C39a9A4D8dE6Bd8B8982E` | 标准市场交易所 |
| Neg Risk CTF Exchange | `0xC5d563A36AE78145C45a50134d48A1215220f80a` | 负风险市场交易所 |
| USDC.e | `0x2791Bca1f2de4661ED88A30C99A7a9449Aa84174` | 抵押品代币 |

#### 机器人检测策略

```
1. 高频交易检测
   - 监控同一地址在短时间内的交易频率
   - 阈值: > 10 笔/分钟 或 < 2 秒间隔

2. Split/Merge 套利检测
   - 当 YES + NO 价格 < $1.00 时的批量 Split 操作
   - 跟踪 splitPosition / mergePositions 合约调用

3. Gas 价格模式
   - 套利机器人通常使用较高 gas price 以确保优先执行
   - 监控 gas price 异常值

4. 关联地址网络
   - 通过资金流向建立地址关联图谱
   - 识别同一运营者的多个机器人地址
```

#### 核心功能

```go
type ChainAnalyzer interface {
    // 实时监控
    WatchExchangeEvents(ctx context.Context, exchangeAddr common.Address) (<-chan ExchangeEvent, error)
    WatchCTFTransfers(ctx context.Context, conditionID [32]byte) (<-chan TransferEvent, error)

    // 机器人检测
    DetectBotActivity(ctx context.Context, opts BotDetectionOpts) ([]*SuspectedBot, error)
    GetAddressProfile(ctx context.Context, addr common.Address) (*AddressProfile, error)

    // 历史分析
    GetMarketTrades(ctx context.Context, marketID string, from, to time.Time) ([]*OnChainTrade, error)
}
```

#### 策略分析维度

| 维度 | 分析内容 |
|------|---------|

| 退出策略 | 持有到期还是提前卖出 |

### 4.6 交易执行引擎 (TradingEngine)

#### 交易流程

```
Signal Detection → Risk Check → Order Construction → EIP-712 Signing → CLOB Submission → Monitoring
```

#### 风险管理参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_position_size` | $100 | 单个市场最大持仓 |
| `max_daily_loss` | $50 | 每日最大亏损 |
| `min_edge` | 5% | 最小边际优势 (扣除 2% 手续费后) |
| `max_open_positions` | 10 | 最大同时持仓市场数 |
| `min_liquidity` | $1,000 | 最低流动性要求 |
| `confirmation_required` | true | MVP 阶段所有交易需人工确认 |

#### 核心功能

```go
type TradingEngine interface {
    // 信号评估
    EvaluateSignal(ctx context.Context, signal TradingSignal) (*TradeDecision, error)

    // 风控检查
    CheckRiskLimits(ctx context.Context, order OrderRequest) error

    // 订单执行
    ExecuteOrder(ctx context.Context, decision TradeDecision) (*OrderResult, error)

    // 持仓管理
    GetOpenPositions(ctx context.Context) ([]*Position, error)
    ClosePosition(ctx context.Context, positionID string) (*OrderResult, error)

    // PnL 统计
    GetPnLReport(ctx context.Context, from, to time.Time) (*PnLReport, error)
}
```

## 9. 外部 API 参考

### 9.1 NOAA / NWS API (解析数据源)

| 项 | 值 |
|---|---|
| **Base URL** | `https://api.weather.gov` |
| **认证** | 无 API Key，需设置 `User-Agent` 请求头 |
| **速率限制** | 未公开，429 时等待 5 秒重试 |
| **数据格式** | GeoJSON (默认)，支持 LD+JSON |

**核心端点:**

| 端点 | 用途 |
|------|------|
| `GET /points/{lat},{lon}` | 获取位置的网格坐标和预报 URL |
| `GET /gridpoints/{wfo}/{x},{y}/forecast` | 12 小时周期预报 (7 天) |
| `GET /gridpoints/{wfo}/{x},{y}/forecast/hourly` | 逐小时预报 (7 天) |
| `GET /gridpoints/{wfo}/{x},{y}` | 原始数值网格数据 |
| `GET /stations/{stationId}/observations/latest` | **最新观测 (市场解析追踪)** |
| `GET /stations/{stationId}/observations` | 历史观测 |
| `GET /alerts/active` | 当前天气警报 |

**NOAA CDO API v2 (历史数据):**

| 项 | 值 |
|---|---|
| **Base URL** | `https://www.ncei.noaa.gov/cdo-web/api/v2/` |
| **认证** | 免费 Token (Header: `token: <TOKEN>`) |
| **速率限制** | 5 req/sec, 10,000 req/day |
| **关键数据集** | GHCND (日数据): TMAX, TMIN, PRCP, SNOW, SNWD |

### 9.2 Open-Meteo API (多模型预报)

| 项 | 值 |
|---|---|
| **Base URL** | `https://api.open-meteo.com/v1/forecast` |
| **认证** | 无 (非商业免费) |
| **速率限制** | 宽松，无官方限制 |

**核心端点:**

| 端点 | 用途 |
|------|------|
| `/v1/forecast` | 多模型预报 (GFS/ECMWF/ICON)，最长 16 天 |
| `/v1/ecmwf` | ECMWF IFS 专用端点，15 天预报 |
| `archive-api.open-meteo.com/v1/archive` | 历史天气 (ERA5, 1940 至今) |
| `historical-forecast-api.open-meteo.com/v1/forecast` | **历史预报存档 (回测关键)** |
| `previous-runs-api.open-meteo.com/v1/forecast` | 预报演变追踪 |

**关键参数:**
```
?latitude=40.7829&longitude=-73.9654
&hourly=temperature_2m,precipitation,wind_speed_10m
&daily=temperature_2m_max,temperature_2m_min,precipitation_sum
&temperature_unit=fahrenheit
&wind_speed_unit=mph
&precipitation_unit=inch
&timezone=America/New_York
&forecast_days=16
```

### 9.3 Visual Crossing (统一历史+预报)

| 项 | 值 |
|---|---|
| **Base URL** | `https://weather.visualcrossing.com/VisualCrossingWebServices/rest/services/timeline/` |
| **认证** | API Key (query param `key=`) |
| **免费额度** | 1,000 records/day |

**核心端点:**

| 端点 | 用途 |
|------|------|
| `/timeline/{location}` | 预报 (未来 15 天) |
| `/timeline/{location}/{startDate}/{endDate}` | 历史数据 (50+ 年) |
| `/timeline/{location}/today` | 今日数据 |

### 9.4 Polymarket APIs

**Gamma API (市场发现):**

| 项 | 值 |
|---|---|
| **Base URL** | `https://gamma-api.polymarket.com` |
| **认证** | 无 |

| 端点 | 用途 |
|------|------|

| `GET /markets/{id}` | 市场详情 |
| `GET /events` | 事件列表 |
| `GET /events/{id}` | 事件详情 |

**CLOB API (交易):**

| 项 | 值 |
|---|---|
| **Base URL** | `https://clob.polymarket.com` |
| **速率限制** | 公开: ~100 req/min; 交易: ~60 orders/min |

| 端点 | 认证 | 用途 |
|------|------|------|
| `GET /midpoint/{tokenId}` | 否 | 中间价 |
| `GET /order-book/{tokenId}` | 否 | 订单簿 |
| `GET /spread/{tokenId}` | 否 | 价差 |
| `GET /last-trade-price/{tokenId}` | 否 | 最新成交价 |
| `POST /order` | 是 | 下单 |
| `DELETE /order/{orderId}` | 是 | 取消订单 |
| `GET /data/orders` | 是 | 我的订单 |
| `GET /data/trades` | 是 | 我的成交 |

**WebSocket 订阅消息:**

**心跳:** 每 10 秒发送 PING。

### C. 天气市场解析规则要点

1. **温度单位**: 市场使用华氏度 (°F)，NWS 报告为官方数据源
2. **时间标准**: NWS 使用当地标准时间 (非夏令时)
3. **解析顺序**: 使用 NWS 的最终报告 (非初步报告)
4. **无数据处理**: 若 NWS 未在截止时间发布数据，市场解析为最低档
5. **精度**: 温度取整数 °F，降水精确到 0.01 英寸
6. **手续费**: Polymarket 对获胜者收取 2% 手续费，套利需 > 2.5-3% 价差才可盈利
