# Sky Alpha Pro 目标架构 DoD 清单

> 版本：v1.0  
> 日期：2026-03-05  
> 用途：定义“按目标架构落地”所需的未实现/未完整实现能力与验收标准

---

## 0. 总体完成定义（Definition of Done）

只有同时满足以下三项，才视为目标架构达成：

1. Agent-First 控制面可稳定运行（非人工兜底常态化）。
2. 系统健康验收与策略价值验收均可通过接口自动判定。
3. 失败可分层归因并可被运维快速定位，无需深挖日志。

---

## 1. P0（必须优先完成）

### P0-1 真正的 Agent 主控会话闭环

目标：

- 以“单轮会话预算”治理，而非仅“调用次数”治理。
- Vertex 控制流程，Go 仅执行工具与风控。

DoD：

- 支持每轮预算：`max_tokens_per_cycle`、`max_tool_calls`、`max_external_requests`、`max_cycle_duration`。
- 超预算时有结构化结束状态与错误码。
- 每轮完整落库：plan、steps、tool_calls、final_decision、no_op_reason。

验收：

- 连续 24h 运行中，无预算失控轮次。
- 每轮可通过接口追溯完整执行轨迹。

---

### P0-2 分布式调度锁（多副本安全）

目标：

- 多副本部署时，`agent_cycle` 与关键 job 不重复执行。

DoD：

- 实现数据库 advisory lock 或 lease lock。
- lock 超时/续约/释放有明确定义。
- 锁竞争状态可观测（含抢锁失败计数）。

验收：

- 双副本压测下，关键 job 单轮仅执行一次。
- 锁异常可自动恢复，不产生长期停摆。

---

### P0-3 机会定义标准化（可执行 edge）

目标：

- 从“理论 edge”升级到“可执行 edge”作为唯一决策依据。

DoD：

- 标准公式：`edge_exec = P_est - P_exec - friction - uncertainty_penalty`。
- `P_exec` 使用可成交价格（bid/ask/depth），非仅 mid/last。
- `friction` 至少含手续费 + 滑点 + spread 成本。

验收：

- 每个候选机会必须落库 `edge_exec` 及构成项。
- 可通过接口统计 `edge_exec > 0` 占比。

---

## 2. P1（高优先级，P0 完成后立即推进）

### P1-1 策略自动进化执行器

目标：

- 让 Agent “可控地进化”，防止随机漂移。

DoD：

- 每轮最多改 1-2 个参数。
- 每次变更必须绑定目标指标（如 candidates_count、edge_exec_positive_rate）。
- 连续 3 轮无改善自动回滚。
- 变更与结果写入 memory（可审计）。

验收：

- 能按接口查看“变更 -> 指标变化 -> 保留/回滚”链路。

---

### P1-2 记忆系统可插拔（External Memory Adapter）

目标：

- 现有记忆可无缝扩展到成熟外部记忆架构。

DoD：

- 定义统一 memory contract（读/写/检索/压缩/过期）。
- 支持本地存储与外部 adapter 双实现。
- 注入策略可配置（topK、时间窗、相似度阈值）。

验收：

- 切换 adapter 后，不影响主流程接口与行为语义。

---

### P1-3 Prompt 版本治理与灰度

目标：

- 让 Prompt 变更可控可回滚可量化。

DoD：

- Prompt 版本化（active/candidate）。
- 支持灰度比例与 A/B 对照。
- 自动记录版本级效果指标（成功率、fallback 率、no_op 分布）。

验收：

- 可在 24h 内完成一次灰度上线与自动回滚演练。

---

## 3. P2（增强项）

### P2-1 价值验收自动化判定器

目标：

- 不依赖人工 SQL，接口自动输出 PASS/NOT PASS。

DoD：

- 分离输出：
  - `health_verdict`
  - `value_verdict`
- 支持 T+0 / T+1 / T+3 判定策略。
- 自动附带 root cause 排名与 next actions。

验收：

- 运维仅通过 `/ops/*` 接口即可完成完整巡检报告。

---

### P2-2 失败分层告警体系

目标：

- 告警能直接指向责任域（配置/数据/策略/执行）。

DoD：

- 标准错误域标签：`failure_domain`。
- 每类告警有单独阈值与处理手册。
- 告警支持抑制策略，避免同类风暴。

验收：

- 任一故障事件在 5 分钟内可归类到唯一主域。

---

## 4. 里程碑建议（实施顺序）

1. M1（P0）：主控闭环 + 分布式锁 + 可执行 edge 标准化。  
2. M2（P1）：自动进化执行器 + 记忆可插拔 + Prompt 灰度。  
3. M3（P2）：自动验收判定 + 失败分层告警完善。  

---

## 5. 非目标（当前阶段不纳入 DoD）

- 前端可视化完整平台（仅保留接口可接入能力）
- 高频交易撮合优化（当前以策略验证优先）
- 多模型协同调度（先稳定单主模型）

---

## 6. 交付物清单（每项必须提供）

每个 DoD 子项交付时需包含：

1. 设计说明（目标、约束、权衡）
2. 接口契约（输入/输出/错误码）
3. 验收脚本或巡检 Prompt
4. 回滚方案
5. 一次真实环境演练结果
