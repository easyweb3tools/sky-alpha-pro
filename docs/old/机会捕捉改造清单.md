# 机会捕捉改造清单（W11+）

## 1. 目标

在保持现有架构不重写的前提下，补齐三个最小可用能力：

1. `market_spec` 最小结构化，避免信号生成时规则口径不完整。
2. `exec_edge` 可执行边际判定，避免仅依赖理论 edge。
3. 信号漏斗落库，快速定位“没信号”的具体断点。

## 2. 数据模型改动

### 2.1 markets 新增字段

- `threshold_f` (`decimal(6,2)`): 从 question 解析出的温度阈值（华氏）。
- `comparator` (`varchar(8)`): 比较符，支持 `ge` / `le`。
- `weather_target_date` (`date`): 对齐天气事件日期，默认 `end_date - 1 day`（UTC）。
- `spec_status` (`varchar(20)`): `ready` / `incomplete`。

### 2.2 signals 新增字段

- `market_price_executable` (`decimal(10,4)`): 可执行价格（按方向和盘口字段选取）。
- `friction_pct` (`decimal(8,4)`): 执行摩擦成本（百分比，含 fee/slippage/spread）。
- `edge_exec_pct` (`decimal(8,4)`): 执行后边际。

### 2.3 新增表 signal_runs

记录每次信号任务漏斗统计：

- `started_at`, `finished_at`, `duration_ms`
- `markets_total`, `spec_ready`, `forecast_ready`, `raw_edge_pass`, `exec_edge_pass`
- `signals_generated`, `skipped`
- `skip_reasons_json`

## 3. 配置改动（signal）

新增：

- `signal.min_edge_exec_pct`（默认 2.0）
- `signal.exec_fee_pct`（默认 0.2）
- `signal.exec_slippage_pct`（默认 0.3）

说明：

- 现有 `signal.min_edge_pct` 继续保留，作为 raw edge 首道过滤。
- `min_edge_exec_pct` 是第二道过滤（可执行边际）。

## 4. 业务逻辑改动

### 4.1 market_spec 生成与兜底

在 signal 生成时按以下顺序构建 spec：

1. 优先读取 `markets` 已有 spec 字段。
2. 缺失时从 question + market_type 推断。
3. 若可推断则回写 markets；否则标记 `spec_status=incomplete` 并跳过。

### 4.2 可执行价格与执行边际

- `raw_edge_pct = (our_estimate - market_price) * 100`
- `market_price_executable`:
  - raw edge >= 0（偏 YES）优先用 `ask_yes`，否则回落 `price_yes`
  - raw edge < 0（偏 NO）优先用 `bid_yes`，否则回落 `price_yes`
- `friction_pct = exec_fee_pct + exec_slippage_pct + spread_pct`
- `edge_exec_pct = ((our_estimate - market_price_executable) * 100)` 按方向扣减摩擦

只在以下条件同时满足时生成信号：

1. `abs(raw_edge_pct) >= min_edge_pct`
2. `abs(edge_exec_pct) >= min_edge_exec_pct`

### 4.3 漏斗统计

每次 `GenerateSignals` 输出并落库：

- `markets_total`
- `spec_ready`
- `forecast_ready`
- `raw_edge_pass`
- `exec_edge_pass`
- `signals_generated`
- `skipped`
- `skip_reasons_json`

## 5. Skip Reason 规范

- `spec_incomplete`
- `city_missing`
- `threshold_missing`
- `latest_price_missing`
- `invalid_market_price`
- `no_forecast`
- `edge_below_raw_threshold`
- `edge_below_exec_threshold`
- `persist_failed`

## 6. 自测清单

1. `go test ./internal/signal ./internal/server ./internal/scheduler` 全部通过。
2. `GenerateSignals` 能返回新增漏斗字段。
3. `signals` 行中新增字段被正确写入。
4. `signal_runs` 有新增记录，且 `skip_reasons_json` 非空时可读。
