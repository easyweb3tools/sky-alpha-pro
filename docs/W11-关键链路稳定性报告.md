# W11 关键链路稳定性报告

更新时间：2026-02-28

## 1. 本周目标

按 W11 计划完成以下事项：

- 失败重试/降级增强
- 关键链路集成测试
- 输出稳定性验证产物

## 2. 本周改动

### 2.1 失败重试增强

- 新增通用重试执行器：`internal/cli/retry.go`
  - 指数退避（`initial_backoff` + `max_backoff`）
  - 可插拔重试判定（`IsRetryable`）
  - 支持重试日志回调（`OnRetry`）
- 接入命令：
  - `market sync`（单次与 `--interval` 模式）
  - `chain scan`
  - `chain watch`（每个周期遇到短时 RPC 问题可自动重试）

### 2.2 关键链路集成测试

- 新增测试：`internal/server/w11_integration_test.go`
- 覆盖链路：
  1. `POST /api/v1/signals/generate`
  2. `POST /api/v1/agent/analyze`
  3. `GET /api/v1/signals`
  4. 校验 `signals` 与 `agent_logs` 持久化结果

测试使用 SQLite 内存库 + 真实 Gin Router 进行端到端调用，不依赖外部网络 API。

### 2.3 重试器单元测试

- 新增测试：`internal/cli/retry_test.go`
- 覆盖场景：
  - 失败后重试并最终成功
  - 非可重试错误立即返回
  - `context canceled` 立即终止

## 4. 风险与后续

当前已完成 W11 的基础稳定性加固，但仍建议在 W12 继续推进：

- 增加 `/readyz` 与外部依赖分级健康检查
